Notes:

Client side validation is when the website prevents a user from inputting something that is in the wrong format.

Form validation can prevent users from submitting passwords that are too short or too long.

It can also be used to ensure that user's input a valid e-mail address or phone number.

We can do client-side validation with a custom JavaScript set up or by using the built-in HTML5 form validation.

We can constrain the length of user inputs and contrain the values of user inputs.

Client-side form validation sometimes requires JavaScript if we want to customize styling and error messages, but it always requires we to think carefully about the user. Always remember to help our users correct the data they provide.

We need to make sure we:

Display explicit error messages.

Be permissive about the input format.

Point out exactly where the error occurs, especially on large forms.

Once we have checked that the form is filled out correctly, the form can be submitted.
